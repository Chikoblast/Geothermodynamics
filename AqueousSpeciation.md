## Step 2: Run the TidyPHREEQC code for calculating aqueous speciation

> Take the aqueous speciation table and input it into **phr_solution()** (see code below). Be careful with the syntax—incorrect formatting will prevent the speciation from being properly imported into TidyPHREEQC calculations. For valid syntax and naming conventions, refer to '**StandardizedNames.md**' and the thermodynamic database files in '**ThermodynamicData**.'

**Code:** Example code demonstrating how to obtain aqueous speciation and mineral saturation indices in raw text format.
```
library(plyr)

#Tidyverse - R package for data science
library(tidyverse)

# devtools::install_github("paleolimbot/tidyphreeqc")
library(tidyphreeqc)

#Database you are using:
phr_use_db_minteq.v4()

#Solution - Initial condition
elm = "Co"

#pH - you can change the range of pH here
pHCode <- c(5,6,7,8,9,10)
pHLen =  1:length(pHCode)

#Phreeqc calculation input

  for (ipH in pHLen){
    #Change here ----->
    sink(sprintf("pH%s_%s_Scenario3.txt",pHCode[ipH],elm))
    phr_run(
      phr_solution(
        #Change here (units are in mM ----->
        "EDTA" = 0.03422,
        "K" = 44.54,
        "Na" = 33.59166, 
        "P" = 35.86,
        "N(3)" = "30.28 as NH4+",
        "N(5)" = "20 as NO3-",
        "S(6)" = "15.16574 as SO4-2", 
        "Mg" = 0.49188, 
        "Cl" = 1.01304,
        "Zn" = 0.00696,
        "Ca" = 0.0068,
        "Fe(2)" = 0.01798,
        "Mo" = 0.01798,
        "Cu(2)" = "0.0008 as Cu+2",
        "Co(2)" = "0.00168 as Co+2",
        "Mn(2)" = "0.00616 as Mn+2",
        "Cyanide" = 0.0097,
        "pH" = pHCode[ipH],
        "temp" = 25)
        ) %>%
        phr_print_output()
        sink()
        }
```
